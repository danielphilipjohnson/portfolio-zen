## Key Terminology: Your React Travel Dictionary

### React Core Concepts

* **React Components** â€“ Reusable pieces of UI, either **Server Components (SCs)** or **Client Components (CCs)**.
    
* **Virtual DOM** â€“ A lightweight **representation of the real DOM** that React uses to optimise updates.
    
* **Hydration** â€“ The process of attaching **event listeners & interactivity** to a server-rendered page.
    

### Server vs. Client Concepts

* **Server Components (SCs)** â€“ Render **on the server**, improve performance, and **don't handle interactivity**.
    
* **Client Components (CCs)** â€“ Render **on the client**, allow **state & interactivity**, but add JavaScript weight.
    
* **Streaming** â€“ Sending Server Components **progressively** instead of waiting for the full page to load.
    

### React Rendering & Performance

* **SSR (Server-Side Rendering)** â€“ Rendering HTML **on the server** before sending it to the browser.
    
* **CSR (Client-Side Rendering)** â€“ Rendering **entirely in the browser**, requiring more JavaScript execution.
    
* **Suspense** â€“ A React feature that **pauses rendering** until data is ready and shows a fallback UI.
    
* **Lazy Loading** â€“ Dynamically **loading components only when needed**, reducing initial page size.
    
* **Reconciliation** â€“ React's diffing algorithm that **updates only changed parts of the UI**, improving efficiency.
    

### Data Fetching & State Management

* **React Query / SWR** â€“ Libraries for **efficient data fetching & caching** in React.
    
* **Context API** â€“ A built-in React feature for **sharing state** between components.
    
* **useState vs. useReducer** â€“ `useState` is for **simple state**, while `useReducer` helps manage **complex logic**.
    

### Debugging & Error Handling

* **Error Boundaries** â€“ Special components that **catch rendering errors** and show a fallback UI.
    
* **DevTools** â€“ Chrome and React Developer Tools that **help inspect and debug components**.
    

### Security & Optimisation

* **Code Splitting** â€“ Breaking JavaScript into smaller chunks **to improve loading speed**.
    
* **Tree Shaking** â€“ Removing **unused code** during the build process.
    
* **Authentication & Authorisation** â€“ Techniques for **securing user data and routes** in React apps.
